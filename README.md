# СОЧНО CRM — десктоп

Персональная CRM для веб-студии: лиды, воронки по каналам трафика, финансы и «сейф»
(пароли / заметки / задачи / документы). Десктопное приложение на **Electron + React + Vite**,
данные хранятся **локально в файле SQLite** и переживают перезапуск.

## Быстрый старт (разработка)

```bash
npm install        # ставит зависимости и пересобирает better-sqlite3 под Electron (postinstall)
npm run dev        # откроется окно Electron с дашбордом и демо-данными
```

Если `better-sqlite3` ругается на версию ABI (после обновления Electron/Node), пересоберите вручную:

```bash
npm run rebuild
```

## Сборка установщика (Windows)

```bash
npm run build      # vite build + electron-builder -> установщик .exe
```

Готовый `.exe` появится в папке **`release/`**. Это обычный NSIS-инсталлятор:
можно выбрать папку установки, создаётся ярлык на рабочем столе.

Для быстрой проверки без инсталлятора — распакованную сборку в `release/win-unpacked`:

```bash
npm run build:dir
```

## Где лежит база данных

Файл `sochno.db` создаётся автоматически в пользовательской папке приложения:

```
Windows: %APPDATA%\sochno-crm\sochno.db
```

- **Экспорт базы** — в подвале сайдбара или меню «Файл → Экспорт базы…»
  (сохраняет копию `sochno.db` в выбранное место, например на флешку).
- **Показать файл базы** — открывает папку с `sochno.db` в проводнике.

## Структура проекта

```
sochno-crm/
├─ package.json
├─ electron-builder.yml
├─ vite.config.ts
├─ index.html
├─ electron/
│  ├─ main.ts          # окно, путь к БД, регистрация IPC, меню, бэкап
│  ├─ preload.ts       # contextBridge -> window.crm
│  └─ db/
│     ├─ index.ts      # подключение better-sqlite3, WAL, схема, seedIfEmpty()
│     ├─ schema.sql    # DDL всех таблиц + индексы
│     └─ repo.ts       # CRUD-функции (репозиторий)
└─ src/
   ├─ main.tsx         # точка входа React
   ├─ App.tsx          # перенос прототипа (компоненты CRM)
   ├─ api.ts           # типизированная обёртка над window.crm
   └─ styles.css       # CSS-блок из прототипа
```

## Архитектура и веб-версия

Весь доступ к данным спрятан за одним интерфейсом — `src/api.ts` (методы `leads/tx/vault/backup`).
React-компоненты обращаются только к нему. Чтобы поднять веб-версию на том же UI, достаточно
подменить реализацию `api.ts`: вместо `window.crm` (IPC) — REST-вызовы к бэкенду
**FastAPI + SQLite** с той же схемой таблиц. Компоненты при этом не меняются.
